# Sum of Subarray Minimums


## Description

Given an array of integers `arr`, find the sum of `min(b)`, where `b` ranges over every (contiguous) subarray of `arr`. 

## Examples

=== "Example 1"

    - Input: `arr = [3, 1, 2, 4]`
    - Output: `17`
    - Explanation: 
    
        - Subarrays are 
            - `[3], [1], [2], [4]`
            - `[3, 1], [1, 2], [2, 4]`
            - `[3, 1, 2], [1, 2, 4]`
            - `[3, 1, 2, 4]`
        - Minimums are 
            - `3`, `1`, `2`, `4`
            - `1`, `1`, `2`
            - `1`, `1`
            - `1`
        - Sum is `17`.

=== "Example 2"

    - Input: arr = `[11, 81, 94, 43, 3]`
    - Output: `444`

=== "Example 2"

    - Input: arr = `[2, 1, 3, 1]`
    - Output: `13`

## Solution

??? "Key Insight"

    The main hurdle here is finding the minimum element in each subarray. But we can flip this problem on it's head:

    > Given a value $v$ at index $i$, we can find subarrays where $v$ is the minimum element.

    To do so, we create two auxiliary arrays:

    - `left[i]`: The index of the **Previous Less Element**. This is the closest element to the left of $i$ that is smaller than `arr[i]`.
    - `right[i]`: The index of the **Next Less Element**. This is the closest element to the right of $i$ that is smaller than `arr[i]`.

    We can use the monotonic stack pattern to populate these two arrays. For a given index $i$, we then know the contiguous subarray where $a_i$ is the smallest element.

    $$
    a_{\text{left}[i] + 1}, \ a_{\text{left}[i] + 2}, \ \dots, \ a_i, \ \dots, \ a_{\text{right}[i] - 2}, \ a_{\text{right}[i] - 1}
    $$

    Because the choice of where the subarray starts is completely independent of where it ends, we multiply the possibilities together (Combinatorics Product Rule). Total subarrays where arr[i] is thus
    
    $$
    (i - left[i]) \times (right[i] - i)
    $$

    !!! note

        Given an array of length $n$, the number of subarrays in it are $\frac{n (n+1)}{2}$. However, if we apply the same logic here, many of those subarray won't even include index $i$. Which is why we can't do the following:

        $$
        \begin{align}
        n &= ((\text{right})[i] - 1) - ((\text{left})[i] + 1) + 1 \\
        \text{count} &= \frac{n(n+1)}{2}
        \end{align}
        $$

??? "Full Solution"

    ```kotlin linenums="1"
    fun sumSubarrayMins(arr: IntArray): Int {
      val stack = ArrayDeque<Int>()
      // index of element smaller than arr[i] in [0, i)
      val left = IntArray(arr.size)
      // index of element smaller than arr[i] in (i, n)
      val right = IntArray(arr.size)

      for ((i, n) in arr.withIndex()) {
        while (stack.isNotEmpty() && arr[stack.last()] >= n) { // (1)
          stack.removeLast()
        }
        left[i] = if (stack.isEmpty()) -1 else stack.last()
        stack.addLast(i)
      }
      
      stack.clear()

      for ((i, n) in arr.withIndex().reversed()) {
        while (stack.isNotEmpty() && arr[stack.last()] > n) { // (2)
          stack.removeLast()
        }
        right[i] = if (stack.isEmpty()) arr.size else stack.last()
        stack.addLast(i)
      }

      var sum = 0L
      for (i in arr.indices) {
        val count = (i - left[i]) * (right[i] - i)
        sum += count * arr[i]
      }
      return sum.toInt()
    }
    ```

    1. Break the symmetry. Eithe the `left` should check `>=` or the `right`. But both can't use `>=` or `>`. This handles the case of duplicate elements.
