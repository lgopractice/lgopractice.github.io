# Implement Queue using Stacks

Implement queue using only two stacks. As a follow-up, see if you can achieve amortized $O(1)$ time complexity?

```kotlin
class MyQueue() {
  fun push(x: Int) {
    // Fill here
  }

  fun pop(): Int {
    // Fill here
  }

  fun peek(): Int {
    // Fill here
  }

  fun empty(): Boolean {
    // Fill here
  }
}
```

## Solutions

??? "First approach"

    $O(1)$ `push` and $O(n)$ `pop`

    We maintain two stacks. `enqueue` is straighforward, we simply always `push` to the first stack.

    $$
    \begin{align}
    \text{stack} = (] \ \ & \ \ \text{aux} = (] \\
    \text{enqueue}(1): \ \ \ \ \ \ \ \ \ \ \text{stack} = (1] \ \ & \ \ \text{aux} = (] \\
    \text{enqueue}(2): \ \ \ \ \ \ \text{stack} = (1, 2] \ \ & \ \ \text{aux} = (] \\
    \text{enqueue}(3): \ \ \text{stack} = (1, 2, 3] \ \ & \ \ \text{aux} = (] \\
    \end{align}
    $$

    `pop` is tricky. We move all elements from $\text{stack} \rightarrow \text{aux}$. `aux.pop()` is our element in question. Finally we close with moving back elements from $\text{aux} \rightarrow \text{stack}$.

    $$
    \begin{align}
    \text{dequeue}(): \ \ \ \ \text{stack} = (1, 2, 3] \ \ & \ \ \text{aux} = (] \\
    \text{stack} = (] \ \ & \ \ \text{aux} = (\fbox{3}, 2, 1] \\
    \text{stack} = (] \ \ & \ \ \text{aux} = (2, 1] \\
    \text{stack} = (1, 2] \ \ & \ \ \text{aux} = (]
    \end{align}
    $$

    here we pop $3$ from $\text{aux}$.

    ```kotlin
    class MyQueue() {

      val stack = java.util.ArrayDeque<Int>()
      val aux = java.util.ArrayDeque<Int>()

      fun push(x: Int) {
        stack.push(x)
      }

      fun pop(): Int {
        stack to aux
        val result = aux.pop()
        aux to stack
        return result
      }

      fun peek(): Int {
        stack to aux
        val result = aux.peek()
        aux to stack
        return result
      }

      fun empty(): Boolean = stack.isEmpty()
    }

    // define our own `to` function.
    infix fun <T> java.util.ArrayDeque<T>.to(dest: java.util.ArrayDeque<T>) {
      while (this.isNotEmpty()) {
        dest.push(this.pop())
      }
    }
    ```

??? "Second approach"

    Previous approach is pretty wasteful. If have a bunch of `pop` one after another, we are shifting elements back and forth between $\text{stack} \leftrightarrow \text{aux}$.

    Approach here is to:

    - always `push` to $\text{stack}$.
    - and always `pop` from $\text{aux}$.


    | operation | $\text{stack}$ | $\text{aux}$ | Comment |
    |-----------|----------------|--------------|---------|
    |           | $(]$           | $(]$         |         |
    | `enqueue(1)` | $(1]$           | $(]$         | always `push` to $\text{stack}$  |
    | `enqueue(2)` | $(2, 1]$           | $(]$         |      |
    | `enqueue(3)` | $(3, 2, 1]$           | $(]$         |   |
    | `dequeue()` | $(]$ | $(1, 2, 3]$<br>$(\fbox{1}, 2, 3]$<br>$(2, 3]$| since $\text{aux}$ is empty, $\text{stack} \rightarrow \text{aux}$<br>`pop` from $\text{aux}$  |
    | `dequeue()` | $(]$ |$(\fbox{2},3]$<br>$(3]$ | since $\text{aux}$ is **not** empty <br>simply `pop` from $\text{aux}$  |

    ```kotlin
    class MyQueue() {

      val stack = java.util.ArrayDeque<Int>()
      val aux = java.util.ArrayDeque<Int>()

      fun push(x: Int) {
        stack.push(x)
      }

      fun pop(): Int {
        if (aux.isEmpty()) {
          stack to aux
        }
        return aux.pop()
      }

      fun peek(): Int {
        if (aux.isEmpty()) {
          stack to aux
        }
        return aux.peek()
      }

      fun empty(): Boolean = stack.isEmpty() && aux.isEmpty()
    }

    infix fun <T> java.util.ArrayDeque<T>.to(dest: java.util.ArrayDeque<T>) {
      while (this.isNotEmpty()) {
        dest.push(this.pop())
      }
    }
    ```
