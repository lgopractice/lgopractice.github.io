# Next Greater Element I

The actual problem is unnecessarily convoluted. Essentially, given an array $nums_2$, we are to find the **next greater element** (NGE) of specified numbers. 

Next greater element of some element $x$ is the first element $e$ to the right of $x$ such that $x < e$.

The elements of $nums_2$ for which we need the NGE is specified in $nums_1$.

```kotlin
fun nextGreaterElement(nums1: IntArray, nums2: IntArray): IntArray {
  // Fill here
}
```

## Example

- Input: $nums_1$ = `[4, 1, 2]` and $nums_2$ = `[1, 3, 4, 2]`
- Output: `[-1, 3, -1]`
- Explanation:
    - $1, 3, \fbox{4}, 2$ there is no element greater than 4.
    - $\fbox{1}, 3, 4, 2$ next greater element is 3.
    - $1, 3, 4, \fbox{2}$ there is no element to its right.

??? Solution

    $nums_1$ contains the elements for which we are being asked to compute NGE. So all we do is compute NGE for all elements.

    ```kotlin
    val stack = java.util.ArrayDeque<Int>()
    val nge = IntArray(nums2.size)

    for ((i, v) in nums2.withIndex().reversed()) {
      while (stack.isNotEmpty() && stack.peek() <= v) {
        stack.pop()
      }
      nge[i] = if (stack.isEmpty()) -1 else stack.peek()
      stack.push(v)
    }
    ```

    The full solution will $nums_1$ handling:

    ```kotlin linenums="1" hl_lines="2 7 15-17"
    fun nextGreaterElement(nums1: IntArray, nums2: IntArray): IntArray {
      val indices = mutableMapOf<Int, Int>()
      val stack = java.util.ArrayDeque<Int>()
      val nge = IntArray(nums2.size) { -1 }

      for ((i, v) in nums2.withIndex().reversed()) {
        indices[v] = i
        while (stack.isNotEmpty() && stack.peek() <= v) {
          stack.pop()
        }
        nge[i] = if (stack.isEmpty()) -1 else stack.peek()
        stack.push(v)
      }

      for ((i, v) in nums1.withIndex()) {
        nums1[i] = nge[indices[v]!!]
      }
      return nums1
    }
    ```
